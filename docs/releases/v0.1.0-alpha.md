# ExplainLens v0.1.0-alpha

**Release date**: 2026-06-05

---

## Overview

This is the first public release of ExplainLens — a local CLI tool that turns papers and
complex texts into visual explainer cards and cartoon storyboards.

The v0.1.0-alpha release is a fully working **local MVP**. It runs entirely offline,
requires no external API keys, and produces clean JSON / Markdown / HTML output.

---

## What Is Included

### Core Pipeline

- **Text parser** — reads `.txt` and `.md` files with encoding detection
- **Chunker** — splits text into semantic chunks with character offsets and section titles
- **Analyzer** — heuristic keyword extraction producing a structured concept map:
  core problem, key concepts, key claims, methods, evidence, limitations, why it matters
- **Planner** — generates an 8-step teaching plan grounded in the concept map
- **Storyboard generator** — maps each teaching step to a cartoon scene with visual metaphor,
  scene composition, characters, caption, and takeaway
- **Image prompt generator** — produces English-language prompts ready for image generation APIs
- **SVG placeholder renderer** — 8 unique geometric SVG illustrations per card
- **HTML renderer** — standalone browser-ready page with hero section, run summary,
  collapsible image prompts, and collapsible source excerpts

### Exports

- `source_chunks.json` — chunked text with character offsets
- `concept_map.json` — structured concept extraction
- `teaching_plan.json` — 8-step teaching plan
- `storyboard.json` — 8 storyboard panels
- `image_prompts.json` — image generation prompts
- `cards.json` — final card data
- `cards.md` — Markdown version
- `cards.html` — standalone HTML preview
- `run_summary.json` — run metadata

## Quick Start

```bash
git clone https://github.com/conanxin/explainlens.git
cd explainlens
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# or: .venv\Scripts\activate  # Windows
pip install -e ".[dev]"
python -m explainlens.cli analyze \
  --input examples/sample_article.txt \
  --output outputs/sample_run
# then open outputs/sample_run/cards.html in a browser
```

## Safety Boundary

- No files are written outside the project directory.
- No external network connections are made.
- No API keys are required or accepted.
- No personal or private documents are included in the repository.
- The `.env` file is never committed (see `.env.example`).

---

## Known Limitations

- Concept extraction uses keyword heuristics, not a language model.
  Quality depends on how well the input document matches common patterns.
- The 8-card structure is fixed in v0.1.0-alpha.
- SVG illustrations are geometric metaphors, not AI-generated art.
- PDF files are not supported yet.
