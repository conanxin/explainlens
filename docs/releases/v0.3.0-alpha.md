## 🎯 What's New in v0.3.0-alpha

### 🖼️ Image Adapter System (Phase 4A)

**Local-first image generation architecture:**

- **Image adapter interface** (`src/explainlens/images/base.py`)
  - Abstract `ImageAdapter` base class
  - Standardized `generate_images()` method
  - Image job tracking (`image_jobs.json`)
  - Manifest disclosure (`image_manifest.json`)

- **`placeholder` adapter** (available)
  - Generates local SVG placeholder images
  - No external API calls
  - Style-aware rendering
  - Safe for CI/testing

- **`fixture` adapter** (experimental)
  - Deterministic SVG for CI/testing
  - Reproducible outputs
  - No randomness

- **CLI integration:**
  - `--image-adapter placeholder|fixture|openai-image`
  - `--image-style clean-cartoon-explainer|whiteboard|storybook|technical-diagram`
  - `--skip-images`
  - `image-adapters` CLI command (list adapters)
  - `image-styles` CLI command (list styles)

### 🔒 OpenAI Image Adapter (Phase 4C — Opt-in Draft)

**Experimental `openai-image` adapter (opt-in only):**

- **Fail-closed by default:**
  - `allow_external_images = False`
  - Requires `--allow-external-images` flag
  - Requires `OPENAI_API_KEY` environment variable

- **No OpenAI SDK dependency:**
  - Uses `urllib.request` for HTTP transport
  - No `import openai`
  - No pip dependency on `openai` package

- **Mock transport for CI:**
  - `run_mock_openai_image_transport()` for testing
  - Zero real API calls in CI
  - Mock returns placeholder SVG

- **Safety guarantees:**
  - No API key in source code
  - No prompt logging to terminal/logs
  - No full source excerpt sent to image adapter
  - Manifest transparency (`uses_external_api: true`)

- **CLI integration:**
  - `--allow-external-images` flag
  - `openai-image` in `--image-adapter` choices
  - Doctor shows `OpenAI Images API: experimental`

### 📊 Image Manifest (`image_manifest.json`)

**New safety disclosure file:**

```json
{
  "image_adapter": "openai-image",
  "image_adapter_version": "openai-image-v0.1",
  "style": "clean-cartoon-explainer",
  "generated_locally": false,
  "external_image_api": true,
  "uses_external_api": true,
  "requires_api_key": true,
  "safety": {
    "prompt_sanitized": true,
    "no_full_text_sent": true
  }
}
```

**Fields:**
- `generated_locally`: `true` for `placeholder`/`fixture`, `false` for `openai-image`
- `external_image_api`: `true` for `openai-image`, `false` otherwise
- `uses_external_api`: matches adapter's `uses_external_api` attribute
- `requires_api_key`: matches adapter's `requires_api_key` attribute

## 🚀 Migration from v0.2.0-alpha

**No breaking changes.**

**New optional flags:**
```bash
# Opt-in to OpenAI image generation (experimental)
export OPENAI_API_KEY="sk-..."
python -m explainlens.cli analyze \
  --input examples/sample_article.txt \
  --output outputs/my_run \
  --image-adapter openai-image \
  --allow-external-images

# Use different image styles
python -m explainlens.cli analyze \
  --input examples/sample_article.txt \
  --output outputs/my_run \
  --image-style storybook

# Skip image generation
python -m explainlens.cli analyze \
  --input examples/sample_article.txt \
  --output outputs/my_run \
  --skip-images
```

## 📋 Known Limitations

**Image generation:**
- `openai-image` adapter is **experimental** and **opt-in only**
- Requires `OPENAI_API_KEY` and `--allow-external-images`
- No image caching yet (future)
- No progress callback for image download (future)

**PDF support:**
- Searchable PDFs only (text-based)
- No OCR for scanned PDFs
- PyMuPDF 1.25+ required

**Providers:**
- `openai` provider is experimental (opt-in via `--allow-external-api`)
- `local-http` provider requires `--allow-local-http`
- `local-fixture` provider is experimental (offline)
